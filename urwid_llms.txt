```text
.. _canvas-cache:

****************
  Canvas Cache
****************

.. currentmodule:: urwid

In Urwid, screen redraws typically update only part of the display. The canvas cache stores unchanged visible canvases to avoid re-rendering widgets unnecessarily.

The :class:`Widget` base class uses metaclass magic to cache results from :meth:`Widget.render`, returning cached canvases on subsequent calls with identical parameters. Invalidate cached state via :meth:`Widget._invalidate`, which forces a redraw on next update.

Similarly, flow widgets' :meth:`Widget.rows` method caches results using the same mechanism; cached rows are returned if matching parameters exist.

**Composite Canvases**  
Container and decoration widgets combine child canvases into nested composite canvases forming a tree rooted at the topmost widget’s :meth:`Widget.render`. These reference children's content/layout, minimizing duplication. When a canvas is invalidated via :meth:`Widget._invalidate`, all direct parents are also removed from cache (cascade-removal), triggering re-rendering on next update. This occurs only once per screen update, enabling efficient batch updates—critical when input processing precedes rendering.

**Cache Lifetime**  
Canvases in the cache are weak references; real references must exist for caching to persist. Urwid’s display modules retain the current topmost canvas, ensuring all visible canvases remain cached. Unused canvases are garbage collected.

**Future Work**  
An update method invalidating only specific screen regions without re-rendering parents would improve efficiency for layout-preserving changes. See mailing list for contributions on display optimizations.
```

.. currentmodule:: urwid

Urwid supports monochrome, 16-color, 88-color, 256-color, and 24-bit (true color) display attributes.  
Support for modes may be enabled via command-line args or interfaces; degradation is graceful.

Palettes in :class:`MainLoop` or directly on screen instances define attributes per mode. Switch modes using:
:meth:`screen.set_terminal_properties() <raw_display.Screen.set_terminal_properties>`, where ``screen`` is a screen instance or :attr:`MainLoop.screen`.

.. seealso::
   :meth:`register_palette() reference <BaseScreen.register_palette>`,

.. _using-display-attributes:

Using Display Attributes
========================

Palette names are used anywhere display attributes are expected. Default: ``None``, rendered using terminal’s default fg/bg.

Use :class:`AttrSpec` for exact fg/bg; requires awareness of current color mode (number of colors).

.. _text-markup:

Text Markup
-----------

:class:`Text` widgets use markup format per :class:`Text class reference <Text>`. Examples:

- `Text(u"a simple string with default attribute")`: text and padding use ``None``.
- `Text(('attr1', u"a string in display attribute attr1"))`: text uses palette's ``'attr1'``, padding defaults.
- `Text([u"a simple string ", ('attr1', u"ending with attr1")])`: first part default, last part ``'attr1'``.
- `Text([('attr1', u"start in attr1 "), ('attr2', u"end in attr2")])`: text uses respective attrs; padding defaults.
- `Text(('attr1', [u"nesting example ", ('attr2', u"inside"), u" outside"]))`: only innermost ``'attr2'`` applies.

.. _display-modules:

*******************
  Display Modules
*******************

.. currentmodule:: urwid

Urwid's display modules abstract screen drawing and user input handling. Choose based on runtime environment.

Typically, select via :class:`MainLoop` constructor:

::

    loop = MainLoop(widget, ..., screen=urwid.display.curses.Screen())
    # Default: urwid.display.raw.Screen() if not specified

Raw vs Curses Display
=====================

- **``display.raw.Screen``**: Pure-Python; sends escape sequences directly. No external deps. Default for :class:`MainLoop`.
- **``display.curses.Screen``**: Uses OS-provided curses/ncurses; optimizes updates, uses termcap. Disables colors on monochrome terminals. Does not support high color (88-/256-/24-bit). Bright foreground without bold unsupported.

| Feature                     | raw       | curses         |
|-----------------------------|-----------|----------------|
| optimized C code            | no        | YES            |
| any terminal compatibility  | no        | YES [1]_       |
| UTF-8 support               | YES       | YES [2]_       |
| bright fg without bold      | YES [3]_  | no             |
| high color support          | YES       | no             |
| mouse dragging              | YES       | no             |
| external event loop         | YES       | no             |

.. [1] if termcap entry exists and ``TERM`` set correctly  
.. [2] if Python linked against wide ncurses  
.. [3] xterm, gnome-terminal only

Other Display Modules
=====================

**CGI Web (``display.web``)**  
Run apps as CGI under Apache. Proof of concept; not production-ready due to security and responsiveness issues. Demonstrated in `tour.py`_, `calc.py`_.

**HTML Screenshot (``display.html_fragment``)**  
:class:`HtmlGenerator` simulates input, captures screen as HTML fragments via :meth:`draw_screen()`. Fragments embeddable in ``<pre>`` tags; text searchable/selectable, responsive to font scaling. See `example screenshots`_.

**LCD Display (``display.lcd``)**  
Base classes for grid-based character displays. Full implementation: :class:`CF635Screen` for Crystal Fontz 635 USB with 6 buttons. Example: `lcd_cf635.py`_.  
.. seealso:: `Urwid on Crystalfontz 635 LCD <http://excess.org/article/2010/03/urwid-crystalfontz-635-lcd/>`

**Setting a Palette**
===================

Pass ``palette`` to :class:`MainLoop`; forwarded to :meth:`BaseScreen.register_palette()`.

Palette: list of attribute definitions with fg/bg per mode (monochrome, normal, high color). Example:

::

    loop = MainLoop(widget, palette=[
        ('headings', 'white,underline', 'black', 'bold,underline'),
        ('body_text', 'dark cyan', 'light gray'),
        ('buttons', 'yellow', 'dark green', 'standout'),
        ('section_text', 'body_text'),  # alias
    ])

See :ref:`display-attributes` for full attribute options.

```text
.. vim: set fileencoding=utf-8:

.. _text-encodings:

***********************
  Encodings Supported
***********************

.. currentmodule:: urwid

Urwid uses a single global text encoding setting, initialized at startup based on the locale. Change it via :meth:`set_encoding`, e.g.:

::

    urwid.set_encoding("UTF-8")

Two modes exist: **Unicode** (for `unicode` strings) and **Pass-through** (for `str` strings).  
- ``txt_a = urwid.Text(u"El Niño")`` → Unicode mode: auto-encoded on display.  
- ``txt_b = urwid.Text("El Niño")`` → Pass-through mode: assumed to be in user’s expected encoding; incorrect mismatch causes "mojibake".

Pass-through is only safe when using encodings that don’t round-trip properly to Unicode, or when fully aware of implications.

**Unicode Support**  
Urwid handles basic character widths for layout and wrapping. No RTL support. Supports all valid Unicode characters under the global encoding, plus common DEC graphics:

```
\u00A3 (£), \u00B0 (°), \u00B1 (±), \u00B7 (·), \u03C0 (π),
\u2260 (≠), \u2264 (≤), \u2265 (≥), \u23ba (⎺), \u23bb (⎻),
\u23bc (⎼), \u23bd (⎽), \u2500 (─), \u2502 (│), \u250c (┌),
\u2510 (┐), \u2514 (└), \u2518 (┘), \u251c (├), \u2524 (┤),
\u252c (┬), \u2534 (┴), \u253c (┼), \u2592 (▒), \u25c6 (◆)
```

Non-UTF-8 encodings send these via alternate character set sequences in compatible terminals.

**Pass-through Support**  
Supported: UTF-8, ISO-8859-*, EUC-JP (JISX 0208 only), EUC-KR, EUC-CN (CN-GB), EUC-TW (CNS 11643 plain 1 only), GB2312, GBK, BIG5, UHC.  
Character width rules:  
- UTF-8: per Unicode standard.  
- ISO-8859-*: all bytes = 1 column wide.  
- Others: high-bit set → half of a 2-column character.

Unimplemented: additional EUC plains.

**Future Work**  
Encoding should be **per-screen (display module)**, not global. Enables mixed encodings across screens—requires modifying widget method signatures to include encoding alongside size and focus.  
Device-specific Unicode mappings also needed (e.g., LCD display with non-standard 8-bit → Unicode mapping), while retaining Unicode text support for device-relevant characters.
```

*************
  Main Loop
*************

.. currentmodule:: urwid

:class:`MainLoop` integrates a :ref:`display module <display-modules>`, widgets, and an :ref:`event loop <event-loops>`. It routes input to widgets, renders the widget tree, and updates the display.

Customize input handling via:
- :meth:`MainLoop.input_filter`: pre-process user input
- :meth:`MainLoop.unhandled_input`: handle unprocessed input

Schedule timed events using:
- :meth:`MainLoop.set_alarm_at`: set alarm at absolute time
- :meth:`MainLoop.set_alarm_in`: set alarm after delay  
- Alarms auto-trigger :func:`MainLoop.draw_screen` post-callback

Exit cleanly via :exc:`ExitMainLoop`. Other exceptions terminate the screen and re-raise.

:class:`MainLoop` is recommended but optional; no dependencies exist in other Urwid components.

**Widget Management**
The top widget is passed to constructor. Change dynamically via:
- ``loop.widget = new_widget`` (for mode/view switching)

Prefer extending widgets for input logic over :meth:`unhandled_input`, as it improves extensibility.

.. _event-loops:

Event Loops
===========

Urwid event loops manage waiting for I/O and timeouts. Support integration with:
- Asyncio_, Twisted_, Glib_, Tornado_, ZMQ_
- Or a ``select``-based loop (default)

Single instance typically suffices, even across multiple :class:`MainLoop` instances.

Add file descriptors via :meth:`watch_file() <SelectEventLoop.watch_file>`, which handles :exc:`ExitMainLoop` and exceptions in Glib_, Twisted_, Tornado_ callbacks.

.. _Asyncio: https://docs.python.org/3/library/asyncio.html
.. _Twisted: http://twistedmatrix.com/trac/
.. _Glib: http://developer.gnome.org/glib/stable/
.. _Tornado: http://www.tornadoweb.org/
.. _ZMQ: https://pyzmq.readthedocs.io/en/latest/

**SelectEventLoop**
-------------------
Default event loop using ``select.select()``.

::

    # Equivalent to:
    loop = urwid.MainLoop(widget)

.. seealso:: :class:`SelectEventLoop reference <SelectEventLoop>`

**AsyncioEventLoop**
--------------------
Integrates with Python's `asyncio` module.

::

    import asyncio
    evl = urwid.AsyncioEventLoop(loop=asyncio.get_event_loop())
    loop = urwid.MainLoop(widget, event_loop=evl)

.. note::
   Use :meth:`run_in_executor` instead of direct executor usage in multithreaded/multiprocess contexts.  
   Matches `asyncio.loop.run_in_executor_`.

.. warning::
   Input uses selectors; only `SelectorEventLoop` supported on Windows.

.. note:: `uvloop` not officially supported.

.. _run_in_executor: https://docs.python.org/3/library/asyncio-eventloop.html#asyncio.loop.run_in_executor
.. _SelectorEventLoop: https://docs.python.org/3/library/asyncio-eventloop.html?highlight=selectoreventloop#asyncio.SelectorEventLoop

.. seealso:: :class:`AsyncioEventLoop reference <AsyncioEventLoop>`

**TwistedEventLoop**
--------------------
Uses Twisted's reactor, emulates :class:`SelectEventLoop` behavior with auto-start/stop on error.

::

    loop = urwid.MainLoop(widget, event_loop=urwid.TwistedEventLoop())

.. note::
   Use `deferToThread_` for threading; avoid `concurrent.futures` executors.

.. _deferToThread: https://docs.twisted.org/en/stable/core/howto/threading.html#getting-results

.. seealso:: :class:`TwistedEventLoop reference <TwistedEventLoop>`

**GLibEventLoop**
-----------------
Uses GLib's event loop; ideal for DBus-integrated apps without Twisted dependency.

::

    loop = urwid.MainLoop(widget, event_loop=urwid.GLibEventLoop())

.. seealso:: :class:`GLibEventLoop reference <GLibEventLoop>`

**TornadoEventLoop**
--------------------
Integrates with Tornado's `IOLoop`.

::

    from tornado.ioloop import IOLoop
    evl = urwid.TornadoEventLoop(IOLoop())
    loop = urwid.MainLoop(widget, event_loop=evl)

.. note::
   Use :meth:`run_in_executor` for multiprocessing/threading; mirrors Tornado’s `IOLoop.run_in_executor_`.

.. seealso:: :class:`TornadoEventLoop reference <TornadoEventLoop>`

**ZMQEventLoop**
----------------
Integrates with 0MQ.

::

    evl = urwid.ZMQEventLoop()
    loop = urwid.MainLoop(widget, event_loop=evl)

.. seealso:: :class:`ZMQEventLoop reference <ZMQEventLoop>`

```text
**Library Overview**  
================  

`urwid` is a console UI library for Python, offering an alternative to direct `curses` usage by automating low-level tasks. Components are loosely coupled and extensible.

- **Display modules**: Handle user input (keystrokes, mouse events), render screen content, map canvas attributes to terminal colors.
- **Widgets**: Modular building blocks; base interface defined in :class:`Widget`. Layout governed by :ref:`widget layout <widget-layout>`.
- **Text handling**: Supports multiple :ref:`text encodings <text-encodings>` and configurable :ref:`text layout <text-layout>` for alignment/wrapping. Custom layouts possible.
- **Display attributes**: Full support for 24-bit/256-color fg/bg, bold, underline, standout; adaptive to terminal capabilities via runtime detection.
- **ListBox**: Powerful scrollable container; controls contents via built-in or custom list walker classes (ideal for long/nested lists with folding).
- **Canvas cache**: Stores weak references to rendered canvases. Maintained by active display modules to avoid recomputation during redraws.
- **Main loop**: Simplifies input handling and screen updates; supports integration with external event loops (:ref:`event-loops <event-loops>`) such as Twisted_ or Glib_.

.. _Glib: http://developer.gnome.org/glib/stable/glib-The-Main-Event-Loop.html  
.. _Twisted: http://twistedmatrix.com/documents/current/core/howto/reactor-basics.html
```

.. _text-layout:

***************
  Text Layout
***************

.. currentmodule:: urwid

Mapping a text string to screen coordinates in a widget is *text layout*. The :class:`Text` widget's default layout (:class:`StandardTextLayout`) supports left, center, or right alignment and wrapping modes: ``'space'`` (default), ``'any'``, ``'clip'``, and ``'ellipsis'``. Wrapping occurs at spaces or any character; clipped text may include an ellipsis.

::

    Text("Showing some different alignment modes", align=...)
    # align='left' (default)
    +----------------+   +------------------------+
    |Showing some    |   |Showing some different  |
    |different       |   |alignment modes         |
    |alignment modes |   +------------------------+
    +----------------+

    # align='center'
    +----------------+   +------------------------+
    |  Showing some  |   | Showing some different |
    |   different    |   |    alignment modes     |
    |alignment modes |   +------------------------+

.. seealso::
   :class:`Text widget reference <Text>`

Custom Text Layouts
===================

The default layout is set via :attr:`Text.layout`. Individual :class:`Text` instances may override this. Custom layouts must extend :class:`TextLayout`, implementing a ``layout()`` method returning text layout structures.

.. seealso::
   :class:`TextLayout reference <TextLayout>`

Text Layout Structures
======================

A text layout structure maps string offsets to screen positions and is used for rendering :class:`Text` widgets and cursor positioning in :class:`Edit`. It consists of a list of line layouts, each representing one row.

Each line layout is a list of tuples:

- A. (*column width*, *start offset*, *end offset*) — displays text segment; column width accounts for zero/double-width characters.
- B. (*space count*, *text offset* or ``None``) — inserts space chars, optionally tied to text offset.
- C. (*column width*, *offset*, *new text*) — inserts arbitrary text (e.g., hyphenation); not used by :class:`StandardTextLayout`.

.. seealso::
   :class:`TextLayout reference <TextLayout>`,
   :class:`StandardTextLayout reference <StandardTextLayout>`

.. _user-input:

**************
  User Input
**************

.. currentmodule:: urwid

User input is parsed by the display module and returned via :meth:`get_input()` or :meth:`get_input_nonblocking()`, as a list. Window resize events are included.

:class:`MainLoop` processes each item in the list, passing it to widget methods :meth:`keypress()` or :meth:`mouse_event()`. Input may be filtered before widget dispatch, or unhandled input caught via functions passed to :class:`MainLoop`'s constructor. On resize, :class:`MainLoop` queries new screen dimensions and updates accordingly.

Multiple keystrokes or mouse events are processed individually per list item.

.. _keyboard-input:

Keyboard Input
==============

Terminal-dependent key transmission; Urwid reports all received keys.

============= =================
Key pressed   Input returned
============= =================
H             ``'h'``
SHIFT+H       ``'H'``
SPACE         ``' '``
ENTER         ``'enter'``
UP            ``'up'``
PAGE DOWN     ``'page down'``
F5            ``'f5'``
SHIFT+F5      ``'shift f5'``
CTRL+SHIFT+F5 ``'shift ctrl f5'``
ALT+J         ``'meta j'``
============= =================

With Unicode :ref:`text encoding <text-encodings>`, non-ASCII characters return as Unicode strings:

=========== ==============
Key pressed Input returned
=========== ==============
é           ``u'é'``
Ж           ``u'Ж'``
カ          ``u'カ'``
=========== ==============

With non-Unicode encodings, input is returned in original encoding:

=========== =========================================
Key pressed Input returned (encoding-dependent)
=========== =========================================
é           ``'é'``
Ж           ``'Ж'``
カ          ``'カ'`` (2 bytes)
=========== =========================================

Urwid does not convert to Unicode to preserve data integrity. Use an input filter for forced Unicode conversion.

.. _mouse-input:

Mouse Input
===========

Input as (*event*, *button*, *x*, *y*) tuple:
- *event*: string, optionally prefixed with ``'shift '``, ``'meta '``, or ``'ctrl'`` if modifier keys held.
- *button*: 1–5 (left, middle, right, scroll up, scroll down).
- *x*, *y*: character coordinates from (0, 0) top-left.

Right mouse and modifiers poorly supported; avoid reliance.

``'mouse press'``
-----------------

=============== ======================
`button`        Mouse button
=============== ======================
1               Left
2               Middle
3               Right
4               Scroll up [#first]_
5               Scroll down [#first]_
=============== ======================

.. [#first] Typically no release event sent

``'mouse release'``
-------------------

*button* = 0 if unknown.

``'mouse drag'``
----------------

Rare; enables tracking mouse movement across cells. Coordinates may be off-screen at edges.

.. _widget-layout:

Widget Layout
=============

Urwid uses widgets to partition screen space, enabling fluid interfaces that adapt to terminal size and font. Rendering proceeds hierarchically:

1. Topmost widget *(a)* renders at full screen dimensions.
2. *(a)* renders child *(b)* up to full screen width/height.
3. *(b)* distributes its available columns among children *(c)*, *(d)*, *(e)*.
4. *(e)* divides its available rows between *(f)* and *(g)*.
5. *(e)* combines canvases from *(f)* and *(g)* → returns to *(b)*.
6. *(b)* combines canvases from *(c)*, *(d)*, *(e)* → returns to *(a)*.
7. *(a)* optionally modifies the canvas → final output.

*(a)*, *(b)*, *(e)* are **container widgets**, responsible for positioning and sizing children. They also track focus: when input flows down the hierarchy:

1. Keypress routed to topmost widget *(a)*.
2. *(a)* → *(b)*.
3. *(b)* → focused child *(e)*.
4. *(e)* → its focused child *(f)*.
5. *(f)* handles or returns keypress.
6. *(e)* may handle if returned from *(f)*.
7. *(b)* may handle if returned from *(e)*.
8. *(a)* may handle if returned from *(b)*.

Box, Flow and Fixed Widgets
===========================

Widget size is measured in screen columns (`maxcol`) and rows (`maxrow`):

- **Box**: container controls both `maxcol`, `maxrow`.
- **Flow**: container sets `maxcol`; widget determines `maxrow` via `.rows()`.
- **Fixed**: widget self-determines both `maxcol`, `maxrow` via `.pack()`.

.. list-table:: Size Determination
   :widths: 20 40 40
   :header-rows: 1

   * - Mode
     - Width
     - Height
   * - ``'box'``
     - Container decides
     - Container decides
   * - ``'flow'``
     - Container decides
     - Widget’s `.rows()`
   * - ``'fixed'``
     - Widget’s `.pack()`
     - Widget’s `.pack()`

Convention: use `maxcol`, `maxrow` for size.  
- Box widgets require `(maxcol, maxrow)` tuple.
- Flow widgets expect `(maxcol,)`.
- Fixed widgets accept `()`.

.. _decoration-widgets:

Decoration Widgets
==================

Alter appearance or position of a single wrapped widget. Accessible via:
- `original_widget`: direct reference to the decorated widget.
- `base_widget`: resolves through chain of decorations; points to self on non-decoration widgets.

Safe for universal use in any context.

.. _container-widgets:

Container Widgets
=================

Divide space among children and manage focus hierarchy. Support nested layouts, where focused child may be multiple levels deep.

**Shared API (preferred)**:
- `container.focus`: read-only → returns currently focused widget; `None` if empty or non-container.
- `container.focus_position`: read/write → position of focused child.  
  - Integer for: `Columns`, `Pile`, `GridFlow`, `Overlay`, `ListBox` with `SimpleListWalker`/`SimpleFocusListWalker`.  
  - String (`'body'`, `'header'`, `'footer'`) for `Frame`.  
  - Custom types via list walker in `ListBox`.  
  → Raises `IndexError` on invalid write or read from empty/non-container widgets. Auto-redraws on update.
- `container.contents`: read-only (mutable in some cases) → mapping/list-like object of `(child, options)` tuples. Supports:
  - `__getitem__(x)`
  - `__setitem__(x, value)`
  - `__delitem__(x)`
  → Triggers redraw upon modification.
  - `Columns`, `Pile`, `GridFlow`: allow assignment from iterable to overwrite contents.
  - All of: `Columns`, `Pile`, `GridFlow`, `Overlay`, `Frame` support item assignment/deletion.
- `container.options(...)`: returns option tuple for container-specific layout. Arguments match container’s `__init__()`.  
  Returns `(str, int, ...)` or `None`; future-proofing via extensible options → **defensive programming required**.

- `container.__getitem__(x)`, alias: `container[x]`:
  - Equivalent to `container.contents[x][0].base_widget`
  - Returns child widget at position *x*, skipping decoration layers.
  - Applies to all containers; skips wrapping decorators like `Padding`.

```python
from __future__ import annotations

import urwid


class SelectablePudding(urwid.Widget):
    _sizing = frozenset((urwid.FLOW,))
    _selectable = True

    def __init__(self) -> None:
        super().__init__()
        self.pudding = "pudding"

    def rows(self, size: tuple[int], focus: bool = False) -> int:
        return 1

    def render(self, size: tuple[int], focus: bool = False) -> urwid.TextCanvas:
        (maxcol,) = size
        num_pudding = maxcol // len(self.pudding)
        pudding = self.pudding
        if focus:
            pudding = pudding.upper()
        return urwid.TextCanvas([pudding.encode("utf-8") * num_pudding], maxcol=maxcol)

    def keypress(self, size: tuple[int], key: str) -> str | None:
        if len(key) > 1:
            return key

        if key.lower() in self.pudding:
            # remove letter from pudding
            n = self.pudding.index(key.lower())
            self.pudding = self.pudding[:n] + self.pudding[n + 1 :]
            if not self.pudding:
                self.pudding = "pudding"
            self._invalidate()
            return None

        return key

```

```python
from __future__ import annotations

import urwid


class CursorPudding(urwid.Widget):
    _sizing = frozenset(["flow"])
    _selectable = True

    def __init__(self):
        super().__init__()
        self.cursor_col = 0

    def rows(self, size: tuple[int], focus: bool = False) -> int:
        return 1

    def render(self, size: tuple[int], focus: bool = False) -> urwid.TextCanvas:
        (maxcol,) = size
        num_pudding = maxcol // len("Pudding")
        cursor = None
        if focus:
            cursor = self.get_cursor_coords(size)
        return urwid.TextCanvas([b"Pudding" * num_pudding], [], cursor=cursor, maxcol=maxcol)

    def get_cursor_coords(self, size: tuple[int]) -> tuple[int, int]:
        (maxcol,) = size
        col = min(self.cursor_col, maxcol - 1)
        return col, 0

    def keypress(self, size: tuple[int], key: str) -> str | None:
        (maxcol,) = size
        if key == "left":
            col = self.cursor_col - 1
        elif key == "right":
            col = self.cursor_col + 1
        else:
            return key
        self.cursor_x = max(0, min(maxcol - 1, col))
        self._invalidate()
        return None

```

```text
Signal Functions  
==================

.. currentmodule:: urwid

The :func:`urwid.\*_signal` functions utilize a shared `Signals` instance for managing signal registration and connections. Do not instantiate `Signals` manually.

- :func:`connect_signal(obj, name, callback, user_arg=None, weak_args=None, user_args=None)`  
  Connects a callback to a signal on `obj`.

- :meth:`Signals.connect(obj, name, callback, user_arg=None, weak_args=None, user_args=None)`  
  Registers a signal handler.

- :func:`disconnect_signal_by_key(obj, name, key)`  
  Removes a connected signal by its unique key.

- :meth:`Signals.disconnect_by_key(obj, name, key)`  
  Unregisters a signal using the provided key.

- :func:`disconnect_signal(obj, name, callback, user_arg=None, weak_args=None, user_args=None)`  
  Disconnects a specific callback from a signal.
```

```rst
******************
  Urwid Tutorial

.. currentmodule:: urwid

Minimal Application
-------------------

Displays ``Hello World`` in top-left corner; runs until *^C*.

.. literalinclude:: minimal.py
   :linenos:

- `txt`: :class:`Text` (flow widget) wraps text to next line, sized by screen width.
- `fill`: :class:`Filler` aligns Text to top, fills rows below with blanks; makes box widget.
- :class:`MainLoop`: renders topmost (box) widget and handles input. Topmost must be a box widget. No input handling here → exit via *^C*.

Global Input
--------------

Displays ``Hello World``, echoes each keypress, exits on *Q*.

.. literalinclude:: input.py
   :linenos:

- :class:`MainLoop` accepts optional `unhandled_input`: called for unhandled keystrokes. Here, all keys pass to `show_or_exit`.
- :exc:`ExitMainLoop`: cleanly exits `MainLoop.run()` when *Q* pressed. Other inputs update Text content.

Display Attributes
------------------

Centers ``Hello World`` with distinct attributes for text, padding, and filler rows; waits keypress before exit.

.. literalinclude:: urwid_attr.py
   :linenos:

- Palette: list of tuples:
  1. Attribute name (str)
  2. FG/BG settings for 16-color mode
  3. BG for 16-color mode
  4. Optional monochrome settings
  5. Optional FG/BG for 88/256-color modes
  6. Optional BG for 88/256-color modes
- :class:`Text` created with `("banner", " Hello World ")`; uses (*attr*, *text*) tuples → text-markup format.
- Multiple attributes: combine tuples in list (see :ref:`text-markup`).
- :class:`AttrMap`: wraps Text with `'streak'`. Maps attribute to children; only unattributed areas (e.g., padding) get new attr.
- Second :class:`AttrMap`: wraps Filler with `'bg'`.

High Color Modes
----------------

Displays ``Hello World`` centered using 256-color mode; exits on *Q*.

.. literalinclude:: highcolors.py
   :linenos:

- Palette defines only high-color FG/BG (ideal for 256-mode). Real apps should define all modes.
- Underlying: :class:`MainLoop` uses `raw_display.Screen`; set terminal to 256-color via `set_terminal_properties()`.
- Top-down widget construction: use placeholders when needed.
- Change topmost widget via `MainLoop.widget = ...`.
- Decoration widgets (e.g., :class:`AttrMap`) have `original_widget` for reassignment.
- :class:`Divider`: creates blank lines, styled with :class:`AttrMap`.
- Container widgets (e.g., :class:`Pile`) use `contents` as list of (*widget*, *options*) tuples; supports `append()`, `options()`.

Question and Answer
-------------------

Asks for name → responds ``Nice to meet you, <name>.``

.. literalinclude:: qa.py
   :linenos:

- :class:`Edit`: extends :class:`Text`; accepts input (cursor control via HOME/END/arrows).
- Custom `QuestionBox` subclasses :class:`Filler`, overrides `keypress()`:
  - All keys except *ENTER* → default `Filler.keypress` → child `Edit.keypress`.
  - *ENTER*: replaces `original_widget` with :class:`Text`; no input handling.
  - Remaining input → `unhandled_input` (`exit_on_q`) → exit on *Q*.
- Pattern: customize decoration/container widgets for input; preferred over global `unhandled_input`.
```

```python
from __future__ import annotations

import urwid


def question():
    return urwid.Pile([urwid.Edit(("I say", "What is your name?\n"))])


def answer(name):
    return urwid.Text(("I say", f"Nice to meet you, {name}\n"))


class ConversationListBox(urwid.ListBox):
    def __init__(self) -> None:
        body = urwid.SimpleFocusListWalker([question()])
        super().__init__(body)

    def keypress(self, size: tuple[int, int], key: str) -> str | None:
        key = super().keypress(size, key)
        if key != "enter":
            return key
        name = self.focus[0].edit_text
        if not name:
            raise urwid.ExitMainLoop()
        # replace or add response
        self.focus.contents[1:] = [(answer(name), self.focus.options())]
        pos = self.focus_position
        # add a new question
        self.body.insert(pos + 1, question())
        self.focus_position = pos + 1
        return None


palette = [("I say", "default,bold", "default")]
urwid.MainLoop(ConversationListBox(), palette).run()
```

from __future__ import annotations

import typing

import urwid

if typing.TYPE_CHECKING:
    from collections.abc import Callable, Hashable, MutableSequence


class ActionButton(urwid.Button):
    def __init__(
        self,
        caption: str | tuple[Hashable, str] | list[str | tuple[Hashable, str]],
        callback: Callable[[ActionButton], typing.Any],
    ) -> None:
        super().__init__("", on_press=callback)
        self._w = urwid.AttrMap(urwid.SelectableIcon(caption, 1), None, focus_map="reversed")


class Place(urwid.WidgetWrap[ActionButton]):
    def __init__(self, name: str, choices: MutableSequence[urwid.Widget]) -> None:
        super().__init__(ActionButton([" > go to ", name], self.enter_place))
        self.heading = urwid.Text(["\nLocation: ", name, "\n"])
        self.choices = choices
        # create links back to ourself
        for child in choices:
            getattr(child, "choices", []).insert(0, self)

    def enter_place(self, button: ActionButton) -> None:
        game.update_place(self)


class Thing(urwid.WidgetWrap[ActionButton]):
    def __init__(self, name: str) -> None:
        super().__init__(ActionButton([" * take ", name], self.take_thing))
        self.name = name

    def take_thing(self, button: ActionButton) -> None:
        self._w = urwid.Text(f" - {self.name} (taken)")
        game.take_thing(self)


def exit_program(button: ActionButton) -> typing.NoReturn:
    raise urwid.ExitMainLoop()


map_top = Place(
    "porch",
    [
        Place(
            "kitchen",
            [
                Place("refrigerator", []),
                Place("cupboard", [Thing("jug")]),
            ],
        ),
        Place(
            "garden",
            [
                Place(
                    "tree",
                    [
                        Thing("lemon"),
                        Thing("bird"),
                    ],
                ),
            ],
        ),
        Place(
            "street",
            [
                Place("store", [Thing("sugar")]),
                Place(
                    "lake",
                    [Place("beach", [])],
                ),
            ],
        ),
    ],
)


class AdventureGame:
    def __init__(self) -> None:
        self.log = urwid.SimpleFocusListWalker([])
        self.top = urwid.ListBox(self.log)
        self.inventory = set()
        self.update_place(map_top)

    def update_place(self, place: Place) -> None:
        if self.log:  # disable interaction with previous place
            self.log[-1] = urwid.WidgetDisable(self.log[-1])
        self.log.append(urwid.Pile([place.heading, *place.choices]))
        self.top.focus_position = len(self.log) - 1
        self.place = place

    def take_thing(self, thing: Thing) -> None:
        self.inventory.add(thing.name)
        if self.inventory >= {"sugar", "lemon", "jug"}:
            response = urwid.Text("You can make lemonade!\n")
            done = ActionButton(" - Joy", exit_program)
            self.log[:] = [response, done]
        else:
            self.update_place(self.place)


game = AdventureGame()
urwid.MainLoop(game.top, palette=[("reversed", "standout", "")]).run()

```python
from __future__ import annotations

import urwid


class CursorPudding(urwid.Widget):
    _sizing = frozenset(["flow"])
    _selectable = True

    def __init__(self):
        super().__init__()
        self.cursor_col = 0

    def rows(self, size: tuple[int], focus: bool = False) -> int:
        return 1

    def render(self, size: tuple[int], focus: bool = False) -> urwid.TextCanvas:
        (maxcol,) = size
        num_pudding = maxcol // len("Pudding")
        cursor = None
        if focus:
            cursor = self.get_cursor_coords(size)
        return urwid.TextCanvas([b"Pudding" * num_pudding], [], cursor=cursor, maxcol=maxcol)

    def get_cursor_coords(self, size: tuple[int]) -> tuple[int, int]:
        (maxcol,) = size
        col = min(self.cursor_col, maxcol - 1)
        return col, 0

    def keypress(self, size: tuple[int], key: str) -> str | None:
        (maxcol,) = size
        if key == "left":
            col = self.cursor_col - 1
        elif key == "right":
            col = self.cursor_col + 1
        else:
            return key
        self.cursor_x = max(0, min(maxcol - 1, col))
        self._invalidate()
        return None
```

```python
from __future__ import annotations

import urwid


class SelectablePudding(urwid.Widget):
    _sizing = frozenset((urwid.FLOW,))
    _selectable = True

    def __init__(self) -> None:
        super().__init__()
        self.pudding = "pudding"

    def rows(self, size: tuple[int], focus: bool = False) -> int:
        return 1

    def render(self, size: tuple[int], focus: bool = False) -> urwid.TextCanvas:
        (maxcol,) = size
        num_pudding = maxcol // len(self.pudding)
        pudding = self.pudding
        if focus:
            pudding = pudding.upper()
        return urwid.TextCanvas([pudding.encode("utf-8") * num_pudding], maxcol=maxcol)

    def keypress(self, size: tuple[int], key: str) -> str | None:
        if len(key) > 1:
            return key

        if key.lower() in self.pudding:
            # remove letter from pudding
            n = self.pudding.index(key.lower())
            self.pudding = self.pudding[:n] + self.pudding[n + 1 :]
            if not self.pudding:
                self.pudding = "pudding"
            self._invalidate()
            return None

        return key
```

```python
import urwid

    from collections.abc import Callable, Hashable, MutableSequence


class ActionButton(urwid.Button):
    def __init__(
        self,
        caption: str | tuple[Hashable, str] | list[str | tuple[Hashable, str]],
        callback: Callable[[ActionButton], typing.Any],
    ) -> None:
        super().__init__("", on_press=callback)
        self._w = urwid.AttrMap(urwid.SelectableIcon(caption, 1), None, focus_map="reversed")


class Place(urwid.WidgetWrap[ActionButton]):
    def __init__(self, name: str, choices: MutableSequence[urwid.Widget]) -> None:
        super().__init__(ActionButton([" > go to ", name], self.enter_place))
        self.heading = urwid.Text(["\nLocation: ", name, "\n"])
        self.choices = choices
        # create links back to ourself
        for child in choices:
            getattr(child, "choices", []).insert(0, self)

    def enter_place(self, button: ActionButton) -> None:
        game.update_place(self)


class Thing(urwid.WidgetWrap[ActionButton]):
    def __init__(self, name: str) -> None:
        super().__init__(ActionButton([" * take ", name], self.take_thing))
        self.name = name

    def take_thing(self, button: ActionButton) -> None:
        self._w = urwid.Text(f" - {self.name} (taken)")
        game.take_thing(self)


def exit_program(button: ActionButton) -> typing.NoReturn:
    raise urwid.ExitMainLoop()


map_top = Place(
    "porch",
    [
        Place(
            "kitchen",
            [
                Place("refrigerator", []),
                Place("cupboard", [Thing("jug")]),
            ],
        ),
        Place(
            "garden",
            [
                Place(
                    "tree",
                    [
                        Thing("lemon"),
                        Thing("bird"),
                    ],
                ),
            ],
        ),
        Place(
            "street",
            [
                Place("store", [Thing("sugar")]),
                Place(
                    "lake",
                    [Place("beach", [])],
                ),
            ],
        ),
    ],
)


class AdventureGame:
    def __init__(self) -> None:
        self.log = urwid.SimpleFocusListWalker([])
        self.top = urwid.ListBox(self.log)
        self.inventory = set()
        self.update_place(map_top)

    def update_place(self, place: Place) -> None:
        if self.log:  # disable interaction with previous place
            self.log[-1] = urwid.WidgetDisable(self.log[-1])
        self.log.append(urwid.Pile([place.heading, *place.choices]))
        self.top.focus_position = len(self.log) - 1
        self.place = place

    def take_thing(self, thing: Thing) -> None:
        self.inventory.add(thing.name)
        if self.inventory >= {"sugar", "lemon", "jug"}:
            response = urwid.Text("You can make lemonade!\n")
            done = ActionButton(" - Joy", exit_program)
            self.log[:] = [response, done]
        else:
            self.update_place(self.place)
```

```python
class CascadingBoxes(urwid.WidgetPlaceholder):
    max_box_levels = 4

    def __init__(self, box: urwid.Widget) -> None:
        super().__init__(urwid.SolidFill("/"))
        self.box_level = 0
        self.open_box(box)

    def open_box(self, box: urwid.Widget) -> None:
        self.original_widget = urwid.Overlay(
            urwid.LineBox(box),
            self.original_widget,
            align=urwid.CENTER,
            width=(urwid.RELATIVE, 80),
            valign=urwid.MIDDLE,
            height=(urwid.RELATIVE, 80),
            min_width=24,
            min_height=8,

            right=(self.max_box_levels - self.box_level - 1) * 3,
            top=self.box_level * 2,
            bottom=(self.max_box_levels - self.box_level - 1) * 2,
        )
        self.box_level += 1

    def keypress(self, size, key: str) -> str | None:
        if key == "esc" and self.box_level > 1:
            self.original_widget = self.original_widget[0]
            self.box_level -= 1
            return None
```

```python
from __future__ import annotations

import urwid


def question():
    return urwid.Pile([urwid.Edit(("I say", "What is your name?\n"))])


def answer(name):
    return urwid.Text(("I say", f"Nice to meet you, {name}\n"))


class ConversationListBox(urwid.ListBox):
    def __init__(self) -> None:
        body = urwid.SimpleFocusListWalker([question()])
        super().__init__(body)

    def keypress(self, size: tuple[int, int], key: str) -> str | None:
        key = super().keypress(size, key)
        if key != "enter":
            return key
        name = self.focus[0].edit_text
        if not name:
            raise urwid.ExitMainLoop()
        # replace or add response
        self.focus.contents[1:] = [(answer(name), self.focus.options())]
        pos = self.focus_position
        # add a new question
        self.body.insert(pos + 1, question())
        self.focus_position = pos + 1
        return None


palette = [("I say", "default,bold", "default")]
urwid.MainLoop(ConversationListBox(), palette).run()
```

```python
from __future__ import annotations

import urwid


class SelectablePudding(urwid.Widget):
    _sizing = frozenset((urwid.FLOW,))
    _selectable = True

    def __init__(self) -> None:
        super().__init__()
        self.pudding = "pudding"

    def rows(self, size: tuple[int], focus: bool = False) -> int:
        return 1

    def render(self, size: tuple[int], focus: bool = False) -> urwid.TextCanvas:
        (maxcol,) = size
        num_pudding = maxcol // len(self.pudding)
        pudding = self.pudding
        if focus:
            pudding = pudding.upper()
        return urwid.TextCanvas([pudding.encode("utf-8") * num_pudding], maxcol=maxcol)

    def keypress(self, size: tuple[int], key: str) -> str | None:
        if len(key) > 1:
            return key

        if key.lower() in self.pudding:
            # remove letter from pudding
            n = self.pudding.index(key.lower())
            self.pudding = self.pudding[:n] + self.pudding[n + 1 :]
            if not self.pudding:
                self.pudding = "pudding"
            self._invalidate()
            return None

        return key

```

```python
from __future__ import annotations

import urwid


class CursorPudding(urwid.Widget):
    _sizing = frozenset(["flow"])
    _selectable = True

    def __init__(self):
        super().__init__()
        self.cursor_col = 0

    def rows(self, size: tuple[int], focus: bool = False) -> int:
        return 1

    def render(self, size: tuple[int], focus: bool = False) -> urwid.TextCanvas:
        (maxcol,) = size
        num_pudding = maxcol // len("Pudding")
        cursor = None
        if focus:
            cursor = self.get_cursor_coords(size)
        return urwid.TextCanvas([b"Pudding" * num_pudding], [], cursor=cursor, maxcol=maxcol)

    def get_cursor_coords(self, size: tuple[int]) -> tuple[int, int]:
        (maxcol,) = size
        col = min(self.cursor_col, maxcol - 1)
        return col, 0

    def keypress(self, size: tuple[int], key: str) -> str | None:
        (maxcol,) = size
        if key == "left":
            col = self.cursor_col - 1
        elif key == "right":
            col = self.cursor_col + 1
        else:
            return key
        self.cursor_x = max(0, min(maxcol - 1, col))
        self._invalidate()
        return None
```

```python
from __future__ import annotations

import urwid


def question():
    return urwid.Pile([urwid.Edit(("I say", "What is your name?\n"))])


def answer(name):
    return urwid.Text(("I say", f"Nice to meet you, {name}\n"))


class ConversationListBox(urwid.ListBox):
    def __init__(self) -> None:
        body = urwid.SimpleFocusListWalker([question()])
        super().__init__(body)

    def keypress(self, size: tuple[int, int], key: str) -> str | None:
        key = super().keypress(size, key)
        if key != "enter":
            return key
        name = self.focus[0].edit_text
        if not name:
            raise urwid.ExitMainLoop()
        # replace or add response
        self.focus.contents[1:] = [(answer(name), self.focus.options())]
        pos = self.focus_position
        # add a new question
        self.body.insert(pos + 1, question())
        self.focus_position = pos + 1
        return None


palette = [("I say", "default,bold", "default")]
urwid.MainLoop(ConversationListBox(), palette).run()
```

```python
    return menu_button([caption, "..."], open_menu)

class CascadingBoxes(urwid.WidgetPlaceholder):
    max_box_levels = 4

    def __init__(self, box: urwid.Widget) -> None:
        super().__init__(urwid.SolidFill("/"))
        self.box_level = 0
        self.open_box(box)

    def open_box(self, box: urwid.Widget) -> None:
        self.original_widget = urwid.Overlay(
            urwid.LineBox(box),
            self.original_widget,
            align=urwid.CENTER,
            width=(urwid.RELATIVE, 80),
            valign=urwid.MIDDLE,
            height=(urwid.RELATIVE, 80),
            min_width=24,
            min_height=8,
            left=self.box_level * 3,
            right=(self.max_box_levels - self.box_level - 1) * 3,
            top=self.box_level * 2,
            bottom=(self.max_box_levels - self.box_level - 1) * 2,
        )
        self.box_level += 1

    def keypress(self, size, key: str) -> str | None:
        if key == "esc" and self.box_level > 1:
            self.original_widget = self.original_widget[0]
            self.box_level -= 1
            return None
```

```python
from __future__ import annotations

import typing

import urwid

if typing.TYPE_CHECKING:
    from collections.abc import Callable, Hashable, MutableSequence


class ActionButton(urwid.Button):
    def __init__(
        self,
        caption: str | tuple[Hashable, str] | list[str | tuple[Hashable, str]],
        callback: Callable[[ActionButton], typing.Any],
    ) -> None:
        super().__init__("", on_press=callback)
        self._w = urwid.AttrMap(urwid.SelectableIcon(caption, 1), None, focus_map="reversed")


class Place(urwid.WidgetWrap[ActionButton]):
    def __init__(self, name: str, choices: MutableSequence[urwid.Widget]) -> None:
        super().__init__(ActionButton([" > go to ", name], self.enter_place))
        self.heading = urwid.Text(["\nLocation: ", name, "\n"])
        self.choices = choices
        # create links back to ourself
        for child in choices:
            getattr(child, "choices", []).insert(0, self)

    def enter_place(self, button: ActionButton) -> None:
        game.update_place(self)


class Thing(urwid.WidgetWrap[ActionButton]):
    def __init__(self, name: str) -> None:
        super().__init__(ActionButton([" * take ", name], self.take_thing))
        self.name = name

    def take_thing(self, button: ActionButton) -> None:
        self._w = urwid.Text(f" - {self.name} (taken)")
        game.take_thing(self)


def exit_program(button: ActionButton) -> typing.NoReturn:
    raise urwid.ExitMainLoop()


map_top = Place(
    "porch",
    [
        Place(
            "kitchen",
            [
                Place("refrigerator", []),
                Place("cupboard", [Thing("jug")]),
            ],
        ),
        Place(
            "garden",
            [
                Place(
                    "tree",
                    [
                        Thing("lemon"),
                        Thing("bird"),
                    ],
                ),
            ],
        ),
        Place(
            "street",
            [
                Place("store", [Thing("sugar")]),
                Place(
                    "lake",
                    [Place("beach", [])],
                ),
            ],
        ),
    ],
)


class AdventureGame:
    def __init__(self) -> None:
        self.log = urwid.SimpleFocusListWalker([])
        self.top = urwid.ListBox(self.log)
        self.inventory = set()
        self.update_place(map_top)

    def update_place(self, place: Place) -> None:
        if self.log:  # disable interaction with previous place
            self.log[-1] = urwid.WidgetDisable(self.log[-1])
        self.log.append(urwid.Pile([place.heading, *place.choices]))
        self.top.focus_position = len(self.log) - 1
        self.place = place

    def take_thing(self, thing: Thing) -> None:
        self.inventory.add(thing.name)
        if self.inventory >= {"sugar", "lemon", "jug"}:
            response = urwid.Text("You can make lemonade!\n")
            done = ActionButton(" - Joy", exit_program)
            self.log[:] = [response, done]
        else:
            self.update_place(self.place)


game = AdventureGame()
urwid.MainLoop(game.top, palette=[("reversed", "standout", "")]).run()
```
